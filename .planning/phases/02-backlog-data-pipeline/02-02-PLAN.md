1. Create src-tauri/src/backlog/commands.rs:

```rust
use tauri::State;
use super::client::BacklogClient;
use super::types::BoardData;

/// Fetch complete board data from Backlog API.
///
/// This is the single IPC entry point for board data (D-02).
/// Parameters use String (not &amp;str) because Tauri async commands
/// cannot use borrowed parameters (Pitfall 4 from RESEARCH).
///
/// Error handling: BacklogError.to_string() produces Japanese messages (D-04).
/// All-or-nothing: any sub-fetch failure returns Err (D-03).
#[tauri::command]
pub async fn fetch_board_data(
    client: State<'_, BacklogClient>,
    host: String,
    api_key: String,
    project_key: String,
    milestone_prefix: String,
    category_ids: Option<Vec<u64>>,
) -> Result<BoardData, String> {
    client
        .fetch_board(
            &host,
            &api_key,
            &project_key,
            &milestone_prefix,
            category_ids.as_deref(),
        )
        .await
        .map_err(|e| e.to_string())
}
```

CRITICAL design decisions reflected:
- Parameters are `String` not `&str` (Tauri async commands cannot use borrowed parameters -- Pitfall 4 from RESEARCH)
- `category_ids: Option<Vec<u64>>` -- per D-06 category filtering at fetch time. Frontend passes None to skip.
- Error mapping: `.map_err(|e| e.to_string())` converts BacklogError to its Japanese Display string (per D-03 + D-04). Addresses review consensus: BacklogError has no Serialize derive, so we convert via Display at this boundary.
- `State<'_, BacklogClient>` -- special Tauri State can use borrowed lifetime in async command. The BacklogClient wraps reqwest::Client for connection pool sharing (addresses review consensus on state semantics).
- Return type `Result<BoardData, String>` -- Tauri commands need String errors. Since BacklogError does NOT derive Serialize (review consensus), String is the correct and only option.

Note on parameter validation: BacklogClient::fetch_board internally calls validate_params() which rejects empty host/api_key/project_key with BacklogError::InvalidInput. The error is converted to String at this boundary, so the frontend receives a Japanese error message.

mod backlog;

use backlog::client::BacklogClient;

#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_http::init())
        .plugin(tauri_plugin_store::Builder::new().build())
        .plugin(tauri_plugin_opener::init())
        .manage(BacklogClient::new())

No Mutex wrapping needed for BacklogClient -- reqwest::Client is Clone and thread-safe. BacklogClient only contains an immutable reqwest::Client reference. The Managed State pattern shares the connection pool across all invocations (review consensus: this is the standard Tauri pattern for HTTP client state).
