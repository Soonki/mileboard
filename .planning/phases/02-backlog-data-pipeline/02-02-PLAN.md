1. Create src-tauri/src/backlog/commands.rs:

```rust
use tauri::State;
use super::client::BacklogClient;
use super::types::BoardData;

#[tauri::command]
pub async fn fetch_board_data(
    client: State<'_, BacklogClient>,
    host: String,
    api_key: String,
    project_key: String,
    milestone_prefix: String,
    category_ids: Option<Vec<u64>>,
) -> Result<BoardData, String> {
    client
        .fetch_board(
            &host,
            &api_key,
            &project_key,
            &milestone_prefix,
            category_ids.as_deref(),
        )
        .await
        .map_err(|e| e.to_string())
}
```

CRITICAL design decisions reflected:
- Parameters are `String` not `&str` (Tauri async commands cannot use borrowed parameters — Pitfall 4 from RESEARCH)
- `category_ids: Option<Vec<u64>>` — per D-06 category filtering at fetch time. Frontend passes None to skip.
- Error mapping: `.map_err(|e| e.to_string())` converts BacklogError to its Japanese Display string (per D-03 + D-04)
- `State<'_, BacklogClient>` — special Tauri State can use borrowed lifetime in async command
- Return type `Result<BoardData, String>` — Tauri commands need String errors or Serialize errors. We use String since BacklogError already has Serialize, but String is simpler and guaranteed.

3. Update src-tauri/src/lib.rs to register BacklogClient as Managed State and add invoke_handler:

```rust
mod backlog;

use backlog::client::BacklogClient;

#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_http::init())
        .plugin(tauri_plugin_store::Builder::new().build())
        .plugin(tauri_plugin_opener::init())
        .manage(BacklogClient::new())
        .invoke_handler(tauri::generate_handler![
            backlog::commands::fetch_board_data
        ])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

IMPORTANT: Preserve ALL existing plugin registrations (.plugin lines). Add `.manage()` before `.invoke_handler()`. The `mod backlog;` declaration must be at the top of the file.

No Mutex wrapping needed for BacklogClient — reqwest::Client is Clone and thread-safe. BacklogClient only contains an immutable reqwest::Client reference.
