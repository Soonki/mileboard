**BacklogClient struct:**
```rust
pub struct BacklogClient {

```

**Constructor:**
```rust
impl BacklogClient {
    pub fn new() -> Self {
        Self {
            http: reqwest::Client::new(),
        }
    }
}
```
reqwest::Client is Clone and uses connection pooling internally — no Mutex needed.

**Private helper — build_url:**
```rust
fn build_url(&self, host: &str, path: &str, api_key: &str, params: &[(&str, &str)]) -> String
```
Constructs `https://{host}/api/v2{path}?apiKey={api_key}&param1=val1&...`. IMPORTANT: never log api_key (security: T-02-01).

**Private helper — check_response:**
```rust
async fn check_response(&self, response: reqwest::Response) -> Result<reqwest::Response, BacklogError>
```
Checks status code: 401 -> BacklogError::Unauthorized, 429 -> BacklogError::RateLimited, 404 -> BacklogError::ProjectNotFound, other non-2xx -> BacklogError::FetchFailed. Calls throttle_if_needed before returning on success.

**Rate limiting — throttle_if_needed (per D-07, Claude's discretion):**
```rust
async fn throttle_if_needed(&self, response: &reqwest::Response)
```
Reads `X-RateLimit-Remaining` header. If value <= 5, reads `X-RateLimit-Reset` (unix timestamp), computes `reset - now + 1` seconds, sleeps via `tokio::time::sleep`. If headers are missing, does nothing (graceful degradation).

**fetch_milestones:**
```rust
pub async fn fetch_milestones(
    &self, host: &str, api_key: &str, project_key: &str, prefix: &str,
) -> Result<Vec<Milestone>, BacklogError>
```
1. GET `/api/v2/projects/{project_key}/versions?apiKey={api_key}`
2. Deserialize as `Vec<Milestone>`
3. Filter: `!m.archived && m.name.starts_with(prefix) && is_milestone_in_range(&m, &start, &end)`
4. Sort by start_date ascending (None sorts last)

**milestone_date_range:**
```rust
fn milestone_date_range() -> (chrono::NaiveDate, chrono::NaiveDate)
```
- start = first day of (today - 1 month): `today.checked_sub_months(Months::new(1)).with_day(1)`
- end = last day of (today + 6 months): `today.checked_add_months(Months::new(7)).with_day(1).pred_opt()`
This gives approximately 7 months of coverage (last month through 6 months ahead).

**is_milestone_in_range:**
```rust
fn is_milestone_in_range(m: &Milestone, start: &NaiveDate, end: &NaiveDate) -> bool
```
Returns true if start_date OR release_due_date falls within [start, end]. Parses date strings as `%Y-%m-%d`. If both dates are None, includes the milestone (defensive — don't hide data).

**fetch_issue_count:**

```rust
async fn fetch_issue_count(
```

**fetch_all_issues (pagination, per Claude's discretion — count-first strategy):**
```rust
pub async fn fetch_all_issues(
    &self, host: &str, api_key: &str, project_key: &str,
    milestone_id: Option<u64>, status_ids: Option<&[u64]>, category_ids: Option<&[u64]>,
) -> Result<Vec<Issue>, BacklogError>
```
1. Call fetch_issue_count to get total
2. Loop: fetch_issues_page with offset += 100 until offset >= total
3. All requests are SEQUENTIAL (no tokio::join! — rate limit constraint per RESEARCH anti-patterns)
4. Return collected Vec<Issue>

**fetch_unassigned_issues (per D-05, D-06):**
```rust
pub async fn fetch_unassigned_issues(

```
1. Compute non-closed status IDs: filter project_statuses where id != 4 (Closed), collect IDs

**fetch_project_statuses:**

```rust
pub async fn fetch_project_statuses(

```

**fetch_board (orchestrator — per D-01, D-02, D-03):**
```rust
pub async fn fetch_board(
    &self, host: &str, api_key: &str, project_key: &str,
    prefix: &str, category_ids: Option<&[u64]>,
```

| Boundary | Description |
|----------|-------------|
| Rust client -> Backlog API | Outbound HTTP to external API with API key in URL |
| Backlog API -> Rust client | Untrusted JSON response data crosses into application |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | client.rs build_url | mitigate | API key used only in URL construction; never included in error messages, log output, or BacklogError variants. build_url is private. |
| T-02-02 | Spoofing | client.rs build_url | accept (Plan 03 scope) | Host URL validated at input boundary (settingsStore). Client trusts host value. SSRF risk is low — desktop app, user-configured host. |
| T-02-03 | Denial of Service | client.rs throttle_if_needed | mitigate | Rate limit header monitoring prevents 429. Sequential-only requests. If remaining <= 5, sleep until reset timestamp. |
| T-02-04 | Tampering | types.rs deserialization | accept | Serde strict typing rejects unexpected shapes. Backlog API is HTTPS-only (reqwest default). Man-in-the-middle risk accepted for desktop app. |
</threat_model>
