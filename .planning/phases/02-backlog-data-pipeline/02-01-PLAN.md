**BacklogClient struct:**
```rust
/// BacklogClient wraps a reqwest::Client for HTTP connection pooling.
/// It is registered as Tauri Managed State so the connection pool is shared
/// across all IPC command invocations. Connection config (host, api_key) is
/// passed per-call because the user may change settings without restarting.
pub struct BacklogClient {
    http: reqwest::Client,
}
```

Addresses review consensus on BacklogClient state semantics: the doc comment explicitly explains WHY it wraps reqwest::Client (connection pooling), WHY it's Managed State (shared pool), and WHY config is per-call (settings may change).

**Constructor:**
```rust
impl BacklogClient {
    pub fn new() -> Self {
        let http = reqwest::Client::builder()
            .timeout(std::time::Duration::from_secs(30))
            .build()
            .unwrap_or_else(|_| reqwest::Client::new());
        Self { http }
    }
}
```
Addresses review concern: request timeout of 30 seconds prevents indefinite hangs. Uses `Client::builder().timeout()` from reqwest. Falls back to default client if builder fails.

**Private helper -- validate_params:**
```rust
fn validate_params(host: &str, api_key: &str, project_key: &str) -> Result<(), BacklogError> {
    if host.trim().is_empty() {
        return Err(BacklogError::InvalidInput("hostが空です".into()));
    }
    if api_key.trim().is_empty() {
        return Err(BacklogError::InvalidInput("APIキーが空です".into()));
    }
    if project_key.trim().is_empty() {
        return Err(BacklogError::InvalidInput("プロジェクトキーが空です".into()));
    }
    Ok(())
}
```
Addresses Codex concern: empty parameters are validated early, not left to flow into HTTP failures.

**Private helper -- build_url:**
```rust
fn build_url(&self, host: &str, path: &str, api_key: &str, params: &[(&str, &str)]) -> String
```
Constructs `https://{host}/api/v2{path}?apiKey={api_key}&param1=val1&...`. IMPORTANT: never log api_key (security: T-02-01).

For repeated query parameters like `statusId[]` and `categoryId[]`, use explicit encoding:
- Each value becomes a separate `statusId[]={id}` pair in the URL
- Example: statusId[]=1&statusId[]=2&statusId[]=3
This addresses Codex concern about exact query encoding for repeated filters.

**Private helper -- check_response:**
```rust
async fn check_response(&self, response: reqwest::Response) -> Result<reqwest::Response, BacklogError>
```
Checks status code: 401 -> BacklogError::Unauthorized, 429 -> BacklogError::RateLimited, 404 -> BacklogError::ProjectNotFound (with project path from URL), other non-2xx -> BacklogError::FetchFailed. Calls throttle_if_needed before returning on success. Checks if error is timeout via `error.is_timeout()` -> BacklogError::Timeout.

**Rate limiting -- throttle_if_needed (per D-07, Claude's discretion):**
```rust
async fn throttle_if_needed(&self, response: &reqwest::Response)
```
Reads `X-RateLimit-Remaining` header. If value <= 5, reads `X-RateLimit-Reset` (unix timestamp), computes `reset - now + 1` seconds, caps at MAX_THROTTLE_SLEEP_SECS = 10, sleeps via `tokio::time::sleep`. If headers are missing or malformed, does nothing (graceful degradation -- addresses review concern about missing headers fallback). The 10-second cap addresses OpenCode concern about long sleep blocking the UI.

Define constant: `const MAX_THROTTLE_SLEEP_SECS: u64 = 10;`
