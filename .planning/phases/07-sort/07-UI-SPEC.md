```
| BoardHeader (existing, 56px height, sticky top:0, z-index:10)                |
+------------------------------------------------------------------------------+
| FilterBar (sticky top:56px, z-index:9)                                       |
| +----------+ +--------+ +----------+ | +------------+ +--+  [chips] [clear] |
| |ステータス ▾| |担当者 ▾| |カテゴリ ▾| | |  ソートなし ▾| |↑ |                 |
| +----------+ +--------+ +----------+ | +------------+ +--+                   |
|                                      sep                                     |
+------------------------------------------------------------------------------+
| Board lanes (scrollable area)                                                |
| [Lane] [Lane] [Lane] [Lane] ...                                             |
+------------------------------------------------------------------------------+
```

- Filter dropdowns and sort controls share the same `.dropdowns` flex container inside FilterBar
- Separator (`|`) divides filter dropdowns from sort controls
- Sort controls: SortDropdown + direction toggle button
- Chips area remains after the dropdowns row (unchanged from Phase 6)
- When sort field is "none" (default), the sort controls are visible but visually subdued (secondary text color)

### Separator

- Element: `<div>` with `role="separator"` and `aria-orientation="vertical"`
- Width: 1px
- Height: 20px
- Background: `--color-border` (`#D1D5DB`)
- Margin: 0 `--space-sm` (8px) on left and right (total 16px gap centered around the line)
- Vertical alignment: centered within the flex row via `align-self: center`

### SortDropdown Layout

```
+------------------------+
| [ソートなし ▾]           |  <-- trigger button (same style as FilterDropdown trigger)
+------------------------+
| +--------------------+ |
| |   ソートなし        | |  <-- selected indicator (no checkbox, highlight bg)
| |   担当者順          | |
| |   期限日順          | |
| +--------------------+ |
+------------------------+
```

- Trigger button: min-width 110px, height 32px, border 1px solid `--color-border`, border-radius `--radius-input` (6px)
- Trigger button padding: `--space-xs` (4px) vertical, `--space-sm` (8px) horizontal
- Trigger button font: `--font-size-body` (14px), `--font-weight-regular` (400)
- Trigger button text color: `--color-text-primary` when sort is active (field !== 'none'), `--color-text-secondary` when field === 'none'
- Trigger button hover: border-color `--color-accent` (same as FilterDropdown)
- Dropdown arrow: Unicode `\u25BE` (small down triangle, same as FilterDropdown)
- Dropdown panel: absolute positioned below trigger, min-width matches trigger, background `--color-surface`
- Dropdown panel border: 1px solid `--color-border`, border-radius `--radius-input` (6px)
- Dropdown panel shadow: `--shadow-dropdown`
- Dropdown panel z-index: 20 (same as FilterDropdown)
- Each option row: padding `--space-xs` (4px) vertical, `--space-sm` (8px) horizontal
- Each option row hover: background `--color-dropdown-hover`
- Selected option: background `--color-dropdown-hover` (persistent, not just on hover) -- no checkbox, background highlight indicates current selection
- Option label font: `--font-size-label` (13px)
- Close behavior: click outside dropdown panel closes it; selecting an option also closes the panel (single-select behavior, unlike FilterDropdown which stays open)

### Direction Toggle Button

```
+----+
| ↑  |  <-- or ↓ depending on direction
+----+
```

- Element: `<button type="button">`
- Width: 32px, height: 32px (square, same height as SortDropdown trigger)
- Border: 1px solid `--color-border`, border-radius `--radius-input` (6px)
- Background: `--color-surface`
- Font: `--font-size-sm` (12px)
- Unicode characters: `\u2191` (upwards arrow) for ascending, `\u2193` (downwards arrow) for descending
- Text color when sort active (field !== 'none'): `--color-text-primary`
- Text color when sort inactive (field === 'none'): `--color-disabled` (`#9CA3AF`)
- Hover: border-color `--color-accent` (same pattern as other trigger buttons)
- Cursor: pointer when sort is active, default when sort is inactive
- Disabled state: when field === 'none', the button is visually muted but still clickable (clicking toggles direction for when a sort field is later selected)

## Interaction Contract

### Sort Selection Flow

1. User clicks the SortDropdown trigger button
2. Dropdown panel appears below the trigger with 3 options
3. User clicks an option -- the sort applies immediately across all lanes (D-08)
4. Dropdown closes after selection (single-select behavior)
5. Trigger button text updates to show the selected sort field name
6. If "ソートなし" is selected, trigger text reverts to "ソートなし" with secondary text color

### Direction Toggle Flow

1. User clicks the direction toggle button
2. Arrow icon flips from `\u2191` to `\u2193` (or vice versa)
3. Sort direction applies immediately across all lanes
4. Default direction is ascending (D-06): assignee A to Z, dueDate early to late

### Sort + Filter Interaction

- Sort is applied AFTER filters (D-09): filter pipeline reduces issues, then sort orders the result
- Changing a filter re-runs both filter and sort in the useMemo pipeline
- Changing a sort re-runs sort on the already-filtered result
- boardStore.data remains raw unfiltered and unsorted (D-10)

### Null Value Behavior (D-07)

- `assignee === null` (unassigned) cards always appear at the END of the sorted list, regardless of ascending or descending direction
- `dueDate === null` (no due date) cards always appear at the END of the sorted list, regardless of ascending or descending direction
- Among null-value cards, secondary sort is by keyId ascending (stable sort)

### Sort Persistence (SORT-04)

- Sort field and direction are saved to plugin-store on every change (fire-and-forget)
- On app startup, sort settings are loaded from plugin-store before initial board render
- If plugin-store data is corrupted or missing, default to field='none', direction='asc'

### DnD Interaction During Sort

- Lane-to-lane DnD remains fully functional while sort is active
- After a DnD move completes, the moved card appears in its sorted position within the target lane (not at the drop position)
- No visual feedback differences for DnD during sort mode

## State Shape Contract

Sort state lives in a new Zustand store (`sortStore`):

```typescript
type SortField = 'none' | 'assignee' | 'dueDate';
type SortDirection = 'asc' | 'desc';

interface SortStoreState {
  field: SortField;       // default: 'none'
  direction: SortDirection; // default: 'asc'

  setField: (field: SortField) => void;
  toggleDirection: () => void;
  loadFromStorage: () => Promise<void>;
}
```

- Default: `{ field: 'none', direction: 'asc' }`
- Sort state IS persisted via plugin-store (SORT-04) -- unlike filter state which is session-only
- Store key in plugin-store: `'sort'` in `settings.json` file (alongside existing `'connection'` key)

## ARIA & Accessibility Contract

| Element | ARIA |
|---------|------|
| FilterBar container (updated) | `role="toolbar"`, `aria-label="フィルタとソート"` |
| Separator | `role="separator"`, `aria-orientation="vertical"` |
| SortDropdown trigger | `aria-haspopup="listbox"`, `aria-expanded="true/false"`, `aria-label="ソート基準"` |
| SortDropdown panel | `role="listbox"`, `aria-activedescendant="{focused-option-id}"` (NO `aria-multiselectable` -- single-select) |
| Each sort option | `role="option"`, `aria-selected="true/false"` |
| Direction toggle button | `aria-label="降順に切り替え"` or `"昇順に切り替え"` (describes action, not current state) |

### Keyboard Accessibility

- SortDropdown trigger: focusable via Tab
- Enter/Space on trigger: opens the dropdown panel
- Arrow Down/Up within panel: navigate options
- Enter/Space on focused option: selects the option and closes the panel
- Escape: closes the dropdown, returns focus to trigger
- Direction toggle: focusable via Tab, Enter/Space toggles direction
- Tab order within FilterBar: filter dropdowns -> separator (skipped) -> SortDropdown -> direction toggle -> chips -> clear button
