## Form Flow States

Per D-11, the form has progressive disclosure:

### State 1: Initial (empty form)
- Host URL input: enabled, empty
- API Key input: enabled, empty
- "接続テスト" button: disabled (host and API key both empty)
- Project dropdown: disabled, placeholder "先に接続テストを実行してください"
- Milestone prefix input: disabled
- "保存" button: disabled

### State 2: Fields entered (host + API key have values)
- "接続テスト" button: enabled (accent color)
- Project dropdown: still disabled
- Milestone prefix input: still disabled
- "保存" button: still disabled

### State 3: Connection testing (loading)
- "接続テスト" button: shows spinner, disabled
- All inputs: read-only during test
- No other visual changes

### State 4a: Connection test success
- Validation message: success (green, checkmark)
- Project dropdown: enabled, loading project list (brief spinner in dropdown)
- Milestone prefix input: enabled
- "保存" button: still disabled (no project selected yet)

### State 4b: Connection test failure
- Validation message: error (red, specific message)
- Project dropdown: remains disabled
- Milestone prefix input: remains disabled
- "保存" button: remains disabled
- User can edit fields and retry

### State 5: Project selected
- "保存" button: enabled (accent color)
- All fields remain editable

### State 6: Save complete
- Full-page mode: Transition to board view (D-02)
- Modal mode: Close modal, board refreshes if settings changed

## Interaction Specifications

### Input Focus Order (Tab sequence)
1. Host URL
2. API Key
3. Connection Test button
4. Project dropdown (when enabled)
5. Milestone prefix (when enabled)
6. Save button (when enabled)

### Keyboard Support
- `Enter` on Connection Test button: triggers connection test
- `Enter` on Save button: triggers save
- `Escape` in modal mode: closes modal
- Tab/Shift+Tab: standard focus navigation

### Disabled Field Behavior
- Visually: reduced opacity (0.5) + `cursor: not-allowed`
- Functionally: not focusable via Tab (use `tabindex="-1"` or `disabled` attribute)

### API Key Visibility Toggle
- Default: masked (`type="password"`)
- Toggle button inside the input field (right-aligned, 32x40px click area)
- Show state: eye icon (U+1F441), input becomes `type="text"`
- Hide state: filled circle (U+25CF), input becomes `type="password"`

### Loading Spinner (CSS-only)
```css
/* Contract: 16px spinner, 2px border, white on accent buttons */
.spinner {
  width: 16px;
  height: 16px;
  border: 2px solid rgba(255,255,255,0.3);
  border-top-color: #FFFFFF;
  border-radius: 50%;
  animation: spin 600ms linear infinite;
}
```
