## Layout Contract

### FilterBar Layout

```
+--------------------------------------------------------------------+
| BoardHeader (existing, 56px height, sticky top:0, z-index:10)      |
+--------------------------------------------------------------------+
| FilterBar (new, auto height, sticky top:56px, z-index:9)           |
| +----------+ +----------+ +----------+  [chip][chip][chip] [clear] |
| | ステータス ▾| | 担当者  ▾ | | カテゴリ ▾|                          |
| +----------+ +----------+ +----------+                             |
+--------------------------------------------------------------------+
| Board lanes (scrollable area, height adjusted)                     |
| [Lane] [Lane] [Lane] [Lane] ...                                   |
+--------------------------------------------------------------------+
```

- FilterBar sits below BoardHeader in the DOM flow
- FilterBar is `position: sticky; top: 56px; z-index: 9` (below BoardHeader z-index:10)
- FilterBar background: `--color-surface` with `border-bottom: 1px solid var(--color-border)`
- FilterBar height: auto (single row when few chips, wraps when many)
- FilterBar min-height: 44px (comfortable touch/click target)
- FilterBar horizontal padding: `--space-md` (16px), matching BoardHeader
- Board height calc adjusts: `calc(100vh - 56px - filterBarHeight)` (use flexbox, not fixed calc)
- When no filters are active, FilterBar still renders with the 3 dropdown buttons visible (no collapse)

### FilterDropdown Layout

```
+------------------------+
| [label ▾]              |  <-- trigger button (border, rounded)
+------------------------+
| +--------------------+ |
| | [ ] Option A       | |  <-- checkbox + label
| | [x] Option B       | |  <-- checked state
| | [ ] Option C       | |
| +--------------------+ |
+------------------------+
```

- Trigger button: min-width 100px, height 32px, border 1px solid `--color-border`, border-radius `--radius-input` (6px)
- Trigger button padding: `--space-xs` (4px) vertical, `--space-sm` (8px) horizontal
- Trigger button font: `--font-size-body` (14px), `--font-weight-regular` (400)
- Trigger button text color: `--color-text-primary` when filters selected, `--color-text-secondary` when none
- Dropdown arrow: Unicode `\u25BE` (small down triangle)
- Dropdown panel: absolute positioned below trigger, min-width matches trigger, max-height 240px with overflow-y auto
- Dropdown panel border: 1px solid `--color-border`, border-radius `--radius-input` (6px)
- Dropdown panel shadow: `--shadow-dropdown`
- Dropdown panel background: `--color-surface`
- Dropdown panel z-index: 20 (above both BoardHeader and FilterBar)
- Each option row: padding `--space-xs` (4px) vertical, `--space-sm` (8px) horizontal
- Each option row hover: background `--color-dropdown-hover`
- Checkbox size: 16px x 16px, border-radius 3px
- Checkbox unchecked: border 1.5px solid `--color-border`, background transparent
- Checkbox checked: background `--color-checkbox-checked`, border transparent, white checkmark (Unicode `\u2713`)
- Gap between checkbox and label text: `--space-sm` (8px)
- Option label font: `--font-size-label` (13px)
- Close behavior: click outside dropdown panel or trigger button closes the panel

- Text: "すべてクリア"
- Style: text-only link (no border, no background)
- Font: `--font-size-sm` (12px), `--font-weight-regular` (400)
- Color: `--color-accent`, hover `--color-accent-hover`
- Cursor: pointer

When a lane has issues in `boardStore.data` but all are filtered out:

- Show a centered message instead of EmptyLane's "課題なし"

- Font: `--font-size-sm` (12px), color `--color-text-secondary`
- Padding: `--space-2xl` (48px) vertical (matches EmptyLane pattern)
- No icon, text only

## Interaction Contract

### Filter Selection Flow

1. User clicks a filter dropdown trigger button
2. Dropdown panel appears below the trigger with checkbox list
3. User checks/unchecks options -- each change applies the filter immediately (FILT-04, D-04)
4. Dropdown stays open for additional selections
5. A FilterChip appears in the chips area for each selected option
6. User clicks outside the dropdown to close it

### Filter Application Logic

- Filters across different axes combine with AND (status AND assignee AND category)
- Filters within the same axis combine with OR (status=Open OR status=InProgress)
- boardStore.data remains unfiltered (D-09); filtering happens in the view layer only
- Filter options are extracted dynamically from all issues in boardStore.data (D-05)
- Options with 0 matching issues are not shown in the dropdown (D-05)
- Assignee dropdown includes a "未割り当て" option for `assignee === null` issues (D-06)

### Dropdown Option Sort Order (Claude's Discretion)

- **Status:** Backlog API `displayOrder` ascending (matches Backlog project settings)
- **Assignee:** Alphabetical by `name` (Japanese locale sort), "未割り当て" always last
- **Category:** Backlog API `displayOrder` ascending, fallback to alphabetical by `name`

### Filter Chip Removal

- Click the remove button (x) on a chip to deselect that single filter option
- The corresponding checkbox in the dropdown unchecks
- "すべてクリア" removes all active filters across all axes

### DnD Interaction During Filtering

- All lanes remain droppable even when showing filtered-empty state (D-07)
- DnD uses IDs from unfiltered boardStore.data for resolution (D-09)
- Moving a card between lanes does not change its filter-axis values (D-08)
- After a card is moved, it remains visible/hidden based on existing filter state

### Keyboard Accessibility

- Dropdown trigger buttons are focusable via Tab
- Enter/Space opens the dropdown
- Arrow keys navigate options within the open dropdown
- Enter/Space toggles the checkbox on the focused option
- Escape closes the dropdown
- Filter chips are focusable; Enter/Delete on a focused chip removes it

## Copywriting Contract

| Element | Copy |
|---------|------|
| Status dropdown label | ステータス |
| Assignee dropdown label | 担当者 |
| Category dropdown label | カテゴリ |
| Clear all button | すべてクリア |
| Unassigned option | 未割り当て |
| Filtered-empty lane | {N}件がフィルタで非表示 |
| Dropdown no-options (edge case) | 選択肢なし |

No empty state heading/body for this phase -- the filter dropdowns are always populated from existing board data. If board has no data, the Board component shows BoardError or BoardSkeleton, not the filter UI.

No error state specific to filtering -- filtering is a pure client-side operation on already-loaded data. Errors are handled at the board data fetch level (existing BoardError component).

No destructive actions in this phase -- clearing filters is non-destructive (restores full view).

## State Shape Contract

Filter state lives in a new Zustand slice or as local state in Board.tsx (planner decides):

```typescript
interface FilterState {
  statusIds: Set<number>;    // selected BacklogStatus.id values
  assigneeIds: Set<number | null>;  // selected BacklogUser.id values, null = unassigned
  categoryIds: Set<number>;  // selected BacklogCategory.id values
}
```

- Default: all Sets empty (no filter applied = show everything)
- Clearing all filters resets all Sets to empty
- Filter state is NOT persisted (per REQUIREMENTS.md Out of Scope: session-only)
