## フィルタ選択UIの形式

| Option | Description | Selected |
|--------|-------------|----------|
| ドロップダウン+チェックボックス | ボタンクリックでチェックボックス付きリストが開く。複数選択可。自前実装でもシンプルに作れる定番パターン | ✓ |
| ポップオーバーパネル | ボタンクリックで広めのパネルが開き、チップクリックでトグル。視認性が高いが実装がやや複雑 | |
| インラインチップトグル | ドロップダウンなし。全選択肢をチップとして常時表示し、クリックでトグル。選択肢が多いとスペースを圧迫する | |

**User's choice:** ドロップダウン+チェックボックス
**Notes:** なし

---

| Option | Description | Selected |
|--------|-------------|----------|
| チェック即反映 | チェックボックスをクリックした瞬間にフィルタが適用される。ドロップダウンは開いたままで追加選択可能。外クリックで閉じる | ✓ |
| 「適用」ボタンで確定 | チェックを付け終わってから「適用」を押すとフィルタが反映される。誤操作は減るがワンステップ多い | |

**User's choice:** チェック即反映
**Notes:** なし

## フィルタ選択肢の生成元

| Option | Description | Selected |
|--------|-------------|----------|
| BoardDataから動的抽出 | 取得済みの全カードからユニークなステータス・担当者・カテゴリを抽出。該当0件の選択肢は自動的に消える。追加APIコール不要 | ✓ |
| プロジェクト設定から取得 | Backlog APIでプロジェクトの全ステータス・全メンバー・全カテゴリを取得。カードにない項目も選べるが、追加APIコール3回必要 | |
| ハイブリッド | ステータス・カテゴリはBoardDataから、担当者はプロジェクトメンバーAPIから。担当者未割当のカードも担当者で絞り込める | |

**User's choice:** BoardDataから動的抽出
**Notes:** なし

---

| Option | Description | Selected |
|--------|-------------|----------|
| 「未割当」選択肢を含める | 担当者ドロップダウンに「未割当」を追加。assignee === nullのカードを絞り込める。スプリントプランニングで「誰にも割り当てられていない課題」を見つけるのに便利 | ✓ |
| 実在の担当者名のみ | 未割当のカードは担当者フィルタでは絞り込めない。シンプルだが、未割当課題を探すには全フィルタ解除が必要 | |

**User's choice:** 「未割当」選択肢を含める
**Notes:** スプリントプランニングでの実用性を重視

## DnD中・移動後のフィルタ挙動

| Option | Description | Selected |
|--------|-------------|----------|
| レーン全体がドロップターゲット | フィルタで表示カードが減っていても、レーン自体は常にドロップ可能。移動はマイルストーン変更なので、フィルタ状態に関係なく機能すべき | ✓ |
| 表示中カード間のみドロップ可 | 非表示カードしかないレーン（「N件非表示」のみ）にはドロップできない。誤操作防止だが、操作性が低下 | |

**User's choice:** レーン全体がドロップターゲット
**Notes:** DnDはマイルストーン間移動でありフィルタ軸（ステータス・担当者・カテゴリ）は変わらないため、フィルタとDnDは独立

---

| Option | Description | Selected |
|--------|-------------|----------|
| 移動先で表示される | マイルストーン移動は常に成功。カードがフィルタ条件外でも移動先レーンに表示される | ✓ |
| 移動後に非表示になる | DnD成功後、フィルタに合致しないカードは移動先レーンでも非表示になる。「N件非表示」のカウントが増える | |

**User's choice:** 移動先で表示される
**Notes:** 実際にはDnDはマイルストーン移動のみでフィルタ軸は変わらないため、この選択は理論上のみ。フィルタ条件の一致/不一致はDnD前後で変化しない
