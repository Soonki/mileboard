## Interaction Contract

### States

| State | Trigger | Visual |
|-------|---------|--------|
| Initial load | App mounts with valid settings | BoardSkeleton (full board skeleton) |
| Loaded | fetchBoardData() resolves | Lanes with cards render |
| Error | fetchBoardData() rejects | BoardError with retry button |
| Reloading | User clicks reload button | Reload icon spins, existing board stays visible |
| Empty lane | Milestone has 0 issues | EmptyLane placeholder text |
| Empty board | All lanes have 0 issues | Normal lanes render with EmptyLane in each |

### Transitions

| From | To | Animation |
|------|-----|-----------|
| Skeleton | Loaded | Fade in (`--transition-normal`, 200ms) |
| Skeleton | Error | Fade in (`--transition-normal`, 200ms) |
| Loaded | Reloading | Reload icon spin only; board content preserved |
| Reloading | Loaded | Seamless update (no flash; Zustand state swap) |
| Reloading | Error | BoardError replaces board content |
| Error | Skeleton | Skeleton replaces error on retry click |

### Reload behavior

1. User clicks reload button
2. Button icon starts spinning (CSS animation: `spin 600ms linear infinite`)
3. Button becomes disabled (`pointer-events: none`, `opacity: 0.5`)
4. Existing board content remains visible during fetch
5. On success: board updates in place (no skeleton flash)
6. On failure: board transitions to error state

### Scroll behavior

- Horizontal: lanes scroll left/right in Board container
- Vertical: cards scroll within each lane independently
- No scroll-snap (free scroll)
- Scrollbar: use native scrollbar (no custom styling)
