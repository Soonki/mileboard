# Requirements Archive: mileboard v1.0

**Defined:** 2026-04-07
**Shipped:** 2026-04-08
**Core Value:** Milestone-to-milestone issue drag-and-drop for fast team planning adjustment

## v1 Requirements

### Connection

- [x] **CONN-01**: User can enter API key, host, and project key and validate the connection
- [x] **CONN-02**: Connection settings are persisted locally and survive app restarts
- [x] **CONN-03**: User can set a milestone filter prefix

### Board Display

- [x] **BOARD-01**: Prefix-matching milestones appear as chronological lanes (last month through 6 months ahead)
- [x] **BOARD-02**: Issues without a milestone appear in an "Unassigned" lane
- [x] **BOARD-03**: Each issue card displays key, title, status badge, assignee, and priority
- [x] **BOARD-04**: Cards are color-coded by status for visual distinction
- [x] **BOARD-05**: Lane headers display total issue count
- [x] **BOARD-06**: Lane headers display member-by-member issue count breakdown

### Drag & Drop

- [x] **DND-01**: Dragging a card between lanes changes the issue's milestone in Backlog
- [x] **DND-02**: Optimistic UI update on drop with rollback to original lane on API failure
- [x] **DND-03**: Multi-milestone issues display in earliest-start-date lane with warning badge and cross-lane DnD disabled

### UX

- [x] **UX-01**: Loading state displayed during initial data fetch
- [x] **UX-02**: Error toast displayed on API failure
- [x] **UX-03**: Clicking a card opens the Backlog issue in the default browser
